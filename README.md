## Backend TypeScript Starter

This project is a minimal Node.js + TypeScript backend starter, configured with strict type checking and a simple entry script.

### Prerequisites

- **Node.js** (recommended: latest LTS)
- **npm** (comes with Node)

### Installation

From the project root (`d:\Working`):

```bash
npm install
```

This installs all dev dependencies including:

- `typescript` – TypeScript compiler
- `ts-node` – run TypeScript directly
- `@types/node` – Node.js type definitions

### TypeScript Configuration

The `tsconfig.json` is configured for backend development with strict type checking:

- **`rootDir`**: `src`
- **`outDir`**: `dist`
- **`module`**: `commonjs`
- **`target`**: `ES2020`
- **`strict`**: `true`
- **`moduleResolution`**: `node`
- **`skipLibCheck`**: `true`
- **`sourceMap`**: `true`

TypeScript only compiles files in `src`, and the compiled JavaScript is written to `dist`.

### Project Structure

- `src/`
  - `index.ts` – main entry point for the backend app
- `dist/` – compiled JavaScript output (generated by `npm run build`)
- `package.json` – npm scripts and dependencies
- `tsconfig.json` – TypeScript compiler options
- `README.md` – this documentation

You can add more backend code under `src/` (for example `src/server.ts`, `src/routes/`, `src/config/`, etc.).

### NPM Scripts

Defined in `package.json`:

- **Build the project**

  ```bash
  npm run build
  ```

  Runs the TypeScript compiler (`tsc`) and outputs JavaScript to `dist/`.

- **Run the compiled app**

  ```bash
  npm start
  ```

  Executes `node dist/index.js`.

- **Run in development with ts-node**

  ```bash
  npm run dev
  ```

  Runs `ts-node src/index.ts`, so you can execute TypeScript directly without a build step (good for development).

### Entry Script (`src/index.ts`)

The starter entry file logs a simple message when the backend starts:

```ts
const startTime = new Date().toISOString();

console.log(`[backend] TypeScript app started at ${startTime}`);
```

You can replace or extend this file to bootstrap your actual backend (e.g. start an HTTP server, load config, connect to a database, etc.).

### Typical Workflows

- **First time setup**

  ```bash
  npm install
  ```

- **Development (quick feedback)**

  ```bash
  npm run dev
  ```

- **Production-style run (compile then run)**

  ```bash
  npm run build
  npm start
  ```
